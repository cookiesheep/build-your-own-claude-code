  Field）是创意编码领域的经典技法——Tyler Hobbs、Ben Fry 
  这些艺术家的作品里经常出现。但它在商业登录页上很少见，这就是差异化。

  关键设计决策：

  1. 不用圆点粒子，用短线段 → 每帧只画 prevXY → currentXY
  的线，配合半透明覆盖产生拖尾 → 看起来像流动的丝线，不像粒子

  3. 鼠标不是推力，是引力井 → 线条被吸引弯曲，不是被弹开 → 更有质感
  4. 错误反馈不是 toast，是冲击波 → 从按钮位置向外扩散的环形扰动 →
  功能和视觉统一

  1. "Make it UNFORGETTABLE" — 它逼我往极端方向想，不做中庸方案

  ---
  三个设计方案

  方案 A：「神经回路」Neural Circuit

  概念：每个 Lab 是一个神经元节点，节点之间有细线连接。Hover 一个 Lab →
  信号脉冲沿着连接线传递到相关 Lab，已完成的 Lab 连接线更亮。

  隐喻：每个 Lab 在你脑中建立新的神经通路，越学越聪明。

  交互：
  - Hover → 从该节点向相邻节点发出一个光点脉冲（沿贝塞尔曲线移动）
  - 已完成 Lab → 连接线持续微弱发光
  - Core Lab → 脉冲是琥珀金色，其他是冷蓝色

  风格：暗色底 + 极细线条 + 偏科技蓝的配色

  优点：隐喻强，交互有意义
  缺点：偏"硬科技感"，可能和琥珀金主色调冲突

  ---
  方案 B：「锻造工坊」Skill Forge

  概念：卡片有微妙的金属纹理质感。Hover 时底部升起暖光，像炉火加热。已完成 Lab
  有「已锻造」的质感印记。Core Lab 有特殊的锤印标记。

  隐喻：你在工坊中一件件锻造 Claude Code 的零件。每完成一个
  Lab，就锻造出一个部件。

  交互：
  - Hover → 底部暖色 glow 从无到有（1.5s 渐入），像金属被加热
  - 卡片边框颜色从冷灰渐变为琥珀金
  - 已完成 → 左上角有一个极小的锻造纹理标记
  - Core Lab → hover 时 glow 更强，有热浪扭曲效果

  风格：工业美学 + 暖色调

  优点：和"Build Your Own"主题高度契合，温暖的工坊感让人踏实
  缺点："锻造"概念可能需要用户脑补理解

  ---
  方案 C：「星图蓝图」Constellation Blueprint ★ 推荐

  概念：背景有极淡的坐标网格（蓝图感），Lab 卡片之间有细线相连形成星座。Hover
  时连接线亮起并轻微脉动。已完成的 Lab 星点更亮，未开始的较暗。Lab
  3（Core）是最亮的那颗星。

  隐喻：你在绘制一张「能力星图」。每完成一个 Lab，就点亮一颗星，星图越来越完整。
  从零散的点到完整的星座——就像从零到拥有完整的 Agent Harness。

  交互：
  1. 连接线动画：Hover 一个 Lab → 它和前后 Lab
  之间的细线亮起，伴随一个从中心向外的微弱脉冲
  2. 亮度梯度：已完成（亮星，带微弱呼吸脉动）> 进行中（中等亮度，有脉冲指示）>
  未开始（暗星，仅轮廓）
  3. 卡片内部：hover 时卡片标题下方出现一句极小的「你会学到什么」副标题（带
  fade-in）
  4. 整体：背景网格在滚动时有极微弱的视差（移动幅度 <
  5px，几乎感知不到但增加了深度感）
  5. 进入动画：保持现有的 staggered 入场，但每张卡片入场时伴随它对应的连接线从 0
   → 1 的绘制动画

  风格：沉静 + 精致 + 有深度。蓝图网格给人"精确、有规划"的感觉，星座给人"宏大、
  有目标"的感觉。两者结合恰好是"系统化学习"的视觉化。

  和现有动效的区别：
  - 首页粒子 → 有机、能量感、感性
  - 登录页流场 → 流动、沉浸、感性
  - 平台页星座蓝图 → 结构、规划、理性 ← 三个页面三种气质

  优点：
  - 和项目主题（渐进式学习、系统构建）高度契合
  - 不和首页/登录页视觉同质化
  - 交互有功能价值（连接线展示 Lab 之间的依赖关系）
  - 沉静但有深度，适合"准备做实验"的心态
  - 技术实现可控（SVG 线条 + CSS 动画，不需要 Canvas）

  缺点：
  - 需要精确定义 Lab 之间的连接关系
  - 卡片间距需要给连接线留空间

  ---
  我的推荐：方案 C「星图蓝图」

  理由：
  1.
  不争不抢：三个页面三种气质（粒子→能量、流场→沉浸、星图→规划），各有各的记忆点
  2. 功能价值：连接线不只是装饰，它实际传达了 Lab 之间的递进关系
  3. 心理暗示：星座的"点亮"隐喻让人有成就感——"我又点亮了一颗星"
  4. 克制的高级感：淡网格 + 细线 + 微脉冲，不浮夸但越看越有细节
  5. 实现风险低：SVG 线条 + CSS transition 就能做，不需要 Canvas/WebGL

  重新构思：三个方案

  方案 A：「代码漂浮」Floating Syntax Blocks ★ 推荐

  概念：页面上漂浮着半透明的、带语法高亮的代码片段。这些片段来自真实的 Claude
  Code 源码（query.ts 的 agent loop、messages.ts 的消息协议、tools/bash.ts
  的工具执行等）。极缓慢地漂浮移动，像水中的浮萍。

  交互：
  - 鼠标靠近 → 该代码片段亮度提升（从 5% → 25% 不透明度），能看清具体代码
  - 点击 → 显示一行小字「Lab 3 会教你实现这段代码」或「这段代码已经在你手中」
  - 代码片段在页面加载时从四面八方缓缓飘入（3-5 秒内布满）

  隐喻：你被代码包围了——这些都是你要学会编写的东西。走近就能看清，走远就若隐若现
  。

  视觉：
  - 代码片段用 JetBrains Mono，12px，带 TypeScript
  语法高亮（关键字=琥珀金，类型=蓝灰，字符串=绿色）
  - 不透明度极低（3-8%），hover 提升到 20-30%
  - 移动速度极慢（每分钟移动 20-30px），像水母漂浮
  - 大小不一（3行到8行不等），角度微倾斜（-5° 到 5°）

  和现有动效的区别：
  - 首页粒子 → 能量、活力、感性
  - 登录页流场 → 流动、沉浸、仪式感
  - 平台页代码漂浮 → 知识感、编码感、你即将进入代码世界

  为什么好：
  - 有功能价值（代码片段就是你要学的内容预览）
  - 极其独特（没见过任何平台这样做）
  - 克制（低透明度 + 慢速，不干扰阅读卡片）
  - 和项目主题完美契合（"Build Your Own Claude Code" — 页面上漂浮的就是 Claude
  Code 的代码）

  ---
  方案 B：「探索光标」Exploration Cursor

  概念：整个平台页面的背景是一个极暗的「代码编辑器」。到处都是极淡的代码行（行号
   + 缩进 + 极低透明度的代码），像在看一份巨大的源文件。鼠标位置处有一个「光标高
  亮带」——鼠标附近的代码行变得可见，远离的几乎看不到。

  交互：
  - 鼠标移动 → 一个 200px 高的高亮区域跟随鼠标，区域内代码可见
  - 点击空白处 → 出现一行打字效果的小注释 // 这里的代码，Lab 3 会教你写
  - 整体像在用一个「代码放大镜」探索一个巨大的源文件

  隐喻：你正在探索一个巨大的代码仓库，鼠标就是你手电筒。哪里亮了，哪里就能看到。

  优点：沉浸感极强，探索感
  缺点：可能干扰卡片阅读，代码行密度需要精确控制

  ---
  方案 C：「代码雨（优雅版）」Elegant Code Drift

  概念：从页面上方缓缓飘落真实的 TypeScript 代码行。不是 Matrix
  那种随机字符，而是有意义的代码：export async function* agentLoop(...)、const 
  response = await client.chat(messages)、tools.register(readFileTool)
  等。每行代码都有语法高亮，极慢地向下漂移，到底部淡出。

  交互：
  - 代码行以不同速度飘落（有的快有的慢，有层次感）
  - 鼠标经过代码行 → 该行短暂停顿 + 亮度提升，像被「抓住」了一瞬间
  - 飘落的代码片段与 Lab 对应：Lab 3 的 agentLoop 代码更频繁出现
  - 点击飘落的代码行 → 代码行变成高亮状态并「贴」到对应的 Lab 卡片上

  隐喻：知识从天而降，你「抓住」哪一段就去学哪一段。

  优点：动感但不浮躁，代码内容有教育价值
  缺点：「代码雨」概念虽然有 Matrix 阴影但确实有人做过类似效果
