## 背景知识

### 什么是 Agent Harness？

你可能用过 Claude Code 或 Cursor Agent — 你用自然语言告诉它做什么，它会自动读代码、写代码、跑命令。这些工具的能力来自两部分：

- **大模型**（Claude, GPT）— 提供智能，占 60%
- **Agent Harness** — 定义工具、管理对话、编排循环，占 40%

"Harness" 的原意是马具——缰绳、鞍具、辔头。马（LLM）有力量，但没有马具，马就只是在草地上乱跑。马具把马的力量**转化为定向的、可控的有用功**。

```
没有 Harness 的 LLM：                   有 Harness 的 LLM（Agent）：

用户说话                                用户说话
  → LLM 生成文字                          → LLM 思考 → "我需要读这个文件"
  → 结束                                  → Harness 帮它读了 → 喂回内容
                                          → LLM 继续思考 → "我要改这行代码"
（这只是聊天机器人）                       → Harness 帮它改了 → 喂回结果
                                          → LLM 判断 → "改好了，完成"

                                        （这才是 Agent）
```

本项目教的就是这个 Harness — 那 40%。

### Claude Code 的真实架构

Claude Code 的完整源码有 416,500 行，但核心 Harness 只有 ~12,000 行（3%）。其中最关键的是 `query.ts` 中的 Agent Loop — 一个 `while(true)` 循环：

1. 调用 LLM
2. LLM 返回：是否要用工具？
3. 如果要 → 执行工具 → 把结果喂回去 → 回到第 1 步
4. 如果不要 → 返回最终回复 → 结束

剥掉错误恢复、上下文压缩、权限控制这些生产级复杂度后，核心逻辑只有约 100 行。**这就是你在 Lab 3 要实现的东西。**
