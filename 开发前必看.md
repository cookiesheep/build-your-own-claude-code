> 用途：给队友快速接手仓库。

---

## 1. 这个仓库是干什么的

`build-your-own-claude-code` 是一个基于真实 Claude Code 源码的渐进式教学平台。

用户体验大致是：

```text
浏览器打开 Lab 页面
→ 在编辑器里改代码
→ 点提交
→ 后端把代码注入 Docker 容器
→ 容器里执行 node build.mjs --lab
→ 浏览器终端连接容器 ttyd
→ 用户运行 node cli.js
→ 看到真实 Claude Code TUI
```

所以这个仓库不是单纯文档仓库，也不是单纯前端仓库，而是：

- 教学文档
- 前端平台
- 后端 API
- Docker 运行环境
- Lab 骨架 / 设计稿

都在一起。

## 3. 根目录里每个目录是干什么的

### [server/](D:/code/build-your-own-claude-code/server)

后端 API。

这里负责：

- anonymous user / token
- session / environment
- submit / progress / workspace
- terminal WebSocket proxy
- Docker container lifecycle
- cleanup / TTL
- E2E regression script

如果你做后端，主要会在这里动。

### [platform/](D:/code/build-your-own-claude-code/platform)

前端平台（Next.js）。

这里负责：

- Lab 页面
- Monaco 编辑器
- xterm.js 终端
- 与后端 API 对接

### [infrastructure/](D:/code/build-your-own-claude-code/infrastructure)

Docker / 运行环境相关。

这里负责：

- `Dockerfile.lab`
- 组装 sister repo 运行底座镜像
- `build-lab-image.ps1`

### [docs/](D:/code/build-your-own-claude-code/docs)

课程文档站源码（MkDocs）。

更偏面向学习者，而不是平台实现本身。

### [labs/](D:/code/build-your-own-claude-code/labs)

Labs 骨架与测试相关目录。

### [shared/](D:/code/build-your-own-claude-code/shared)

共享类型定义。

### [internal/](D:/code/build-your-own-claude-code/internal)

**项目内部设计稿和工作流核心目录。**

如果你是队友或新会话 AI，这里最值得读。
