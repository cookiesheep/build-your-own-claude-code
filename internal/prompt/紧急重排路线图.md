## 一、竞争分析

### 竞品：codecrafters-io/build-your-own-claude-code

- CodeCrafters 平台出品（已有成熟用户基础）
- 形式：命令行交互式教程（类似 rustlings）
- 不提供 Web IDE / Docker 容器 / 真实 TUI 体验
- **我们的差异化**：浏览器内写代码 → 注入 Docker 容器 → 看到真实 Claude Code TUI 运行

### 竞争策略

1. **速度 > 完备**：先让公开页面上线，哪怕功能不完整
2. **差异化展示**：首页突出"真实 TUI + Docker 容器"这一竞品没有的核心优势
3. **尽早占领 GitHub 星标 / 社区认知**

## 三、PR-P1.1 认证系统设计

### 核心决策

| 问题 | 决策 | 理由 |
|------|------|------|
| 认证方式 | 用户名 + 密码 | Leader 手动创建账号给队友/内测者 |
| 谁能注册 | 仅管理员可创建账号 | 防止公开访问者随意注册 |
| 会话管理 | 签名 cookie (httpOnly) | 安全，防 XSS |
| 匿名访问 | 保留 /platform 和 / 可匿名浏览 | 首页和 Lab 选择页不需要登录 |
| 受保护路由 | /lab/:id 需要登录 | 实验工作台消耗容器资源，需授权 |
| 数据库 | SQLite users 表 | 与现有 sessions/progress 同库 |

### 后端改动

```
新增：
  server/src/routes/auth.ts        — POST /api/auth/login, POST /api/auth/logout, GET /api/auth/me
  server/src/middleware/auth.ts     — 认证中间件（cookie 校验）
  server/src/db/database.ts        — users 表 + 密码哈希

修改：
  server/src/index.ts              — 注册 auth 路由 + 中间件
  server/src/routes/session.ts     — 加入 auth 校验
  server/src/routes/environment.ts — 加入 auth 校验
  server/src/routes/submit.ts      — 加入 auth 校验

新增管理工具：
  server/src/scripts/create-user.ts — CLI 创建用户脚本
```

### 前端改动

```
新增：
  platform/src/app/login/page.tsx       — 登录页面
  platform/src/components/AuthGuard.tsx — 客户端认证守卫
  platform/src/lib/auth.ts             — 认证状态管理

修改：
  platform/src/components/Navbar.tsx    — 显示登录状态/用户名/登出按钮
  platform/src/app/lab/[id]/page.tsx    — 包裹 AuthGuard
  platform/src/app/platform/page.tsx    — 未登录时提示"登录后开始实验"
```

## 四、PR-P1.2 API Key 管理

### 核心决策

| 问题 | 决策 | 理由 |
|------|------|------|
| 默认 Key | 服务端环境变量 `DEFAULT_API_KEY` | 所有用户共享，减少 Key 暴露 |
| 用户自带 Key | 用户可在设置面板填入自己的 Key | 灵活，高级用户可自行管理 |
| Key 存储 | SQLite user_settings 表，AES 加密 | 安全 |
| 注入方式 | 容器创建时通过 ENV 注入 | 与现有架构一致 |
| 优先级 | 用户 Key > 默认 Key | 用户自带时优先使用 |

### 后端改动

```
新增：
  server/src/routes/settings.ts           — GET/PUT /api/settings/api-key
  server/src/services/encryption.ts       — AES 加密/解密工具
  server/src/db/database.ts              — user_settings 表

修改：
  server/src/services/container-manager.ts — 创建容器时注入 ANTHROPIC_API_KEY ENV
  server/src/index.ts                     — 注册 settings 路由
```

### 前端改动

```
新增：
  platform/src/components/SettingsModal.tsx — 设置弹窗（API Key 输入）
  platform/src/lib/settings.ts             — 设置 API 客户端

修改：
  platform/src/components/Navbar.tsx       — 加入设置按钮
  platform/src/components/LabWorkspace.tsx — 显示当前使用 Key 来源
```

### 详细提示词

见 `internal/API_KEY_PROMPT.md`
