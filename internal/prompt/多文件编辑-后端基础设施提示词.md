## 项目背景

**Build Your Own Claude Code (BYOCC)** 教学平台。学习者通过 6 个 Lab 逐步实现 Claude Code 的核心模块。

当前系统只支持**单文件编辑**：一个 Lab 一个 `code` 字符串，注入到容器的 `/workspace/src/query-lab.ts`，构建时只替换 `query.js`。

本任务改为**多文件编辑模式**：每个 Lab 可以有多个可编辑文件，支持 Lab 0 的 6 文件场景。

### 核心设计决策

**1. 命名约定：`{原名}-lab{N}.{ext}`**

这是串联所有环节的唯一约定：

| 容器原始文件 | Lab 文件（学习者编辑） | Lab | 构建时替换 |
|---|---|---|---|
| `src/query.ts` | `src/query-lab3.ts` | 3 | `query-lab3.ts → query.js` |
| `src/messages.ts` | `src/messages-lab1.ts` | 1 | `messages-lab1.ts → messages.js` |
| `src/components/LogoV2/Clawd.tsx` | `Clawd-lab0.tsx` | 0 | `Clawd-lab0.tsx → Clawd.js` |
| `src/main.tsx` | `src/main-lab0.tsx` | 0 | `main-lab0.tsx → main.js` |

一条约定串联三个环节：
1. **文件树**：`LAB_FILES` 列出 `*-lab{N}.*` 文件
2. **代码注入**：`injectFiles()` 写到 `/workspace/${path}`
3. **构建**：`build.mjs --lab N` 自动发现并替换所有 `*-lab{N}.*` 文件

**2. 编辑不需要容器**

用户应该先阅读代码、编写代码，最后才启动容器运行。

- **可编辑文件内容**：来自 `LAB_FILES` 配置的骨架代码 + workspace 已保存的内容
- **只读文件内容**：来自容器（需要容器运行，已由 file-read-api 实现）
- 容器只在 **submit → inject → build → run** 时需要

#### 自动生成文件树

删除手写的 `FILE_TREE` 嵌套数组。改为从 `LAB_FILES[labId]` 自动生成：

```typescript
/** 从扁平文件列表生成嵌套文件树 */
export function buildLabFileTree(labId: number): FileTreeNode[] {
  const files = LAB_FILES[labId] ?? [];
  return buildTreeFromPaths(files.map(f => f.path));
}

function buildTreeFromPaths(paths: string[]): FileTreeNode[] {
  const root: FileTreeNode[] = [];
  const dirMap = new Map<string, FileTreeNode>();

  for (const path of paths) {
    const parts = path.split('/');
    let current = root;

    for (let i = 0; i < parts.length; i++) {
      const name = parts[i];
      const nodePath = parts.slice(0, i + 1).join('/');
      const isFile = i === parts.length - 1;

      // 查找已有节点
      const existing = current.find(n => n.name === name);
      if (existing) {
        if (!isFile) {
          current = existing.children!;
        }
        continue;
      }

      const node: FileTreeNode = isFile
        ? { name, type: "file", path: nodePath }
        : { name, type: "directory", path: nodePath, children: [] };

      current.push(node);
      if (!isFile) {
        current = node.children!;
      }
    }
  }

  // 排序：目录在前，文件在后，同类型按名称排序
  function sortNodes(nodes: FileTreeNode[]): FileTreeNode[] {
    return nodes.sort((a, b) => {
      if (a.type !== b.type) return a.type === "directory" ? -1 : 1;
      return a.name.localeCompare(b.name);
    }).map(node => {
      if (node.children) {
        return { ...node, children: sortNodes(node.children) };
      }
      return node;
    });
  }

  return sortNodes(root);
}
```

#### 新增函数

```typescript
/**
 * 注入多个文件到容器
 *
 * @param sessionId - 会话 ID
 * @param files - 文件路径 → 代码内容映射
 */
export async function injectFiles(
  sessionId: string,
  files: Record<string, string>
): Promise<void> {
  const { container } = await getContainerOrThrow(sessionId);

  for (const [filePath, code] of Object.entries(files)) {
    if (!code || !filePath) continue;

    // 安全校验：路径必须在 /workspace/ 下
    if (filePath.includes('..') || filePath.startsWith('/')) {
      throw new Error(`Invalid file path: ${filePath}`);
    }

    const targetFile = `/workspace/${filePath}`;
    const encodedCode = Buffer.from(code, 'utf8').toString('base64');

    const { exitCode, output } = await runExecCommand(container, [
      'bash', '-c',
      `mkdir -p $(dirname ${shellQuote(targetFile)}) && printf '%s' '${encodedCode}' | base64 -d > ${shellQuote(targetFile)}`,
    ]);

    if (exitCode !== 0) {
      throw new Error(
        `Failed to inject ${filePath}: ${output || 'No output'}`
      );
    }
  }
}
```

**推荐做法**：保持列名 `code` 不变，内容改为 JSON 字符串。读取时自动检测格式：

```typescript
export function getWorkspace(
  userId: string,
  labNumber: number
): { files: Record<string, string>; updatedAt: string | null } | null {
  const database = getDb();
  const row = database
    .prepare('SELECT code, updated_at FROM code_snapshots WHERE user_id = ? AND lab_number = ?')
    .get(userId, labNumber) as { code: string; updated_at: string } | undefined;

  if (!row) {
    return { files: {}, updatedAt: null };
  }

  // 兼容旧格式：单个字符串 vs JSON map
  let files: Record<string, string>;
  try {
    const parsed = JSON.parse(row.code);
    if (typeof parsed === 'object' && parsed !== null && !Array.isArray(parsed)) {
      files = parsed as Record<string, string>;
    } else {
      // 旧格式：纯字符串，放入默认路径
      files = { 'src/query-lab.ts': row.code };
    }
  } catch {
    // 旧格式：不是 JSON，是纯代码字符串
    files = { 'src/query-lab.ts': row.code };
  }

  return { files, updatedAt: row.updated_at };
}

export function saveWorkspace(
  userId: string,
  labNumber: number,
  files: Record<string, string>
): { files: Record<string, string>; updatedAt: string } {
  const database = getDb();
  const filesJson = JSON.stringify(files);

  database
    .prepare(`
      INSERT INTO code_snapshots (user_id, lab_number, code, updated_at)
      VALUES (?, ?, ?, datetime('now'))
      ON CONFLICT(user_id, lab_number) DO UPDATE SET
        code = excluded.code,
        updated_at = datetime('now')
    `)
    .run(userId, labNumber, filesJson);

  const result = getWorkspace(userId, labNumber);
  return {
    files: result?.files ?? files,
    updatedAt: result?.updatedAt ?? new Date().toISOString(),
  };
}
```

**注意**：保留旧的 `upsertCodeSnapshot` 和 `getCodeSnapshot` 函数不删（submit.ts 的 `upsertCodeSnapshot(access.user.id, labNumber, code)` 还在用）。新函数 `getWorkspace` 和 `saveWorkspace` 是新的接口。以后可以统一。

const labMode = process.argv.includes('--lab');
if (labMode) {

  const labIndex = process.argv.indexOf('--lab');

  const labSuffix = `-lab${labNumber}`;

  // 1. 扫描所有 *-lab{N}.ts 和 *-lab{N}.tsx
  const labFiles = [];
  function findLabFiles(dir) {

    // src/query-lab3.ts → target: src/query.ts
    const targetRelPath = relPath.replace(`${labSuffix}`, '');

        target: 'node18',

          'MACRO.VERSION_CHANGELOG': JSON.stringify(''),
        },
        logLevel: 'warning',
      });
    } catch (err) {
      console.error(`  ✗ Failed to compile ${relPath}:`, err.message);
      continue;

    const labOutFile = join(labOutDir, basename(relPath).replace(/\.tsx?$/, '.js'));

}

全部完成后手动验证：

1. **LAB_FILES 配置**：
   ```bash
   cd platform && node -e "
     const { LAB_FILES, LAB_EDITABLE_FILES } = require('./src/lib/file-tree-data.ts');
     console.log('Lab 3 files:', LAB_FILES[3]);
     console.log('Lab 0 files:', LAB_FILES[0].length, 'files');
   "
   ```

2. **Workspace API 兼容性**：
   - PUT `{ code: "old format" }` → GET 返回 `{ files: { "src/query-lab.ts": "old format" } }`
   - PUT `{ files: { "a.ts": "code" } }` → GET 返回 `{ files: { "a.ts": "code" } }`

3. **Submit API 兼容性**：
   - POST `{ code: "...", sessionId, labNumber }` → 成功注入到 `/workspace/src/query-lab.ts`
   - POST `{ files: { "src/query-lab3.ts": "..." }, sessionId, labNumber }` → 成功注入

4. **build.mjs**：
   - `node build.mjs --lab 3` → 输出显示扫描到 `query-lab3.ts`，替换 `query.js`
