## 项目背景

**BYOCC** 教学平台，运行在 Win11 桌面服务器上，通过 Cloudflare Tunnel 暴露到公网。

### 目标域名

`byocc.cc`（HTTP → Cloudflare Tunnel → 本地 Next.js + Express）

### 当前开发架构

```
浏览器 → http://localhost:3000 (Next.js)
              ↓ rewrite /api/*
              → http://127.0.0.1:3001 (Express)
                    ↓ Dockerode
                    → Docker 容器 (ttyd 端口随机映射)
```

Next.js 的 `next.config.ts` 已配置了 rewrite：
```typescript
rewrites: [{ source: "/api/:path*", destination: "http://127.0.0.1:3001/api/:path*" }]
```

### 生产目标架构

```
浏览器 → https://byocc.cc (Cloudflare Tunnel HTTPS)
              ↓
              → http://127.0.0.1:3000 (Next.js production)
                    ↓ rewrite /api/*
                    → http://127.0.0.1:3001 (Express)
                          ↓ Dockerode
                          → Docker 容器
```

**关键**：浏览器只和 `byocc.cc` 通信。Cloudflare Tunnel 终止 HTTPS，转发 HTTP 到本地。Express 不直接对外。

## 开发任务

### 任务 1：后端读取 lab-files.json 的生产路径修复

**文件**：`server/src/services/lab-workspace.ts`

当前问题：`readFileSync` 路径 `../../../platform/src/lib/lab-files.json` 假设开发布局，生产环境不存在。

**推荐方案：构建时生成 TypeScript 常量**

新增脚本 `server/scripts/generate-lab-config.mjs`：

```javascript
// 读取 platform/src/lib/lab-files.json
// 生成 server/src/services/lab-files-generated.ts
// 导出: export const LAB_FILES: Record<number, LabFileDef[]> = { ... };
```

修改 `lab-workspace.ts`：

```typescript
// 改前（运行时读文件）：
const labFilesConfig = JSON.parse(readFileSync(...));

// 改后（编译时生成）：
import { LAB_FILES } from './lab-files-generated.js';
```

在 `server/package.json` 的 `build` script 中加入生成步骤：

```json
"build": "node scripts/generate-lab-config.mjs && tsc"
```

这样：
- 开发和生产走同一条路径（import TypeScript 常量）
- 没有运行时文件 IO
- `lab-files.json` 仍然是唯一 source of truth
- 构建时自动同步

### 任务 2：生产环境变量配置

**文件**：`server/.env`（本地文件，不提交）

创建 `server/.env.example` 作为模板，列出所有需要替换的变量。

需要修改的环境变量：

```bash
# === 必须替换 ===

# 认证密钥 — 用 openssl rand -hex 32 生成
BYOCC_AUTH_SECRET=<openssl rand -hex 32 的输出>
SERVER_SESSION_SECRET=<openssl rand -hex 32 的输出>

# 加密密钥 — 用 openssl rand -hex 16 生成
# ⚠️ 如果已存在加密过的 API Key，改了这个 Key 会导致解密失败
# 如果没有用户数据，可以放心换。如果有，先清空 user_settings 表
ENCRYPTION_KEY=<openssl rand -hex 16 的输出>

# === 生产域名相关 ===

# CORS：只允许生产域名
CORS_ORIGINS=https://byocc.cc

# Cookie Secure：HTTPS 下必须 true
BYOCC_COOKIE_SECURE=true

# 监听地址：Cloudflare Tunnel 连本地，127.0.0.1 即可
HOST=127.0.0.1
PORT=3001

# === 可选调整 ===

# 匿名登录：生产环境建议关闭，强制注册登录
# 如果要开放体验模式可以保持 true
BYOCC_ANONYMOUS_AUTH_ENABLED=false

# 容器 TTL：120 分钟，根据需要调整
BYOCC_CONTAINER_TTL_MINUTES=120
```

创建 `.env.example` 文件放在 `server/` 目录：

```bash
# === 认证 ===
BYOCC_AUTH_SECRET=             # openssl rand -hex 32
SERVER_SESSION_SECRET=         # openssl rand -hex 32
ENCRYPTION_KEY=                # openssl rand -hex 16（改了会解密失败旧 Key）

# === 网络与安全 ===
HOST=127.0.0.1
PORT=3001
CORS_ORIGINS=https://byocc.cc  # 生产域名，逗号分隔多个
BYOCC_COOKIE_SECURE=true        # HTTPS 必须 true

# === 认证模式 ===

# === 容器 ===

BYOCC_CONTAINER_TTL_MINUTES=120
```

### 任务 3：Next.js 生产配置

**文件**：`platform/next.config.ts`

确认当前 rewrite 配置在生产 `next start` 模式下工作正常。Next.js rewrites 在 production 模式下同样生效。

检查是否需要额外配置：

```typescript
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  async rewrites() {
    return [
      {
        source: "/api/:path*",
        destination: "http://127.0.0.1:3001/api/:path*",
      },
    ];
  },
};

export default nextConfig;
```

**需要确认的点**：
1. `NEXT_PUBLIC_API_URL` 环境变量 — 如果前端 API 客户端使用了这个变量，在生产环境中应该**留空**（让 rewrite 处理，浏览器请求 `https://byocc.cc/api/*`，Next.js rewrite 到 Express）。如果设了值会导致绕过 rewrite。
2. 检查 `platform/src/lib/api.ts` 中的 `apiUrl()` 函数 — 确认 `API_BASE` 为空时使用相对路径（`/api/...`），让 Next.js rewrite 生效。

### 任务 4：终端 WebSocket 连接的生产化

**这是部署中最容易出问题的环节。**

当前开发环境：浏览器直连 `ws://localhost:PORT/ws`（Docker 容器的 ttyd 端口）。

生产环境：浏览器必须通过 `wss://byocc.cc` 连接。

**检查 `server/src/index.ts` 的 `setupWebSocketProxy(server)`**：
- 确认它代理了 WebSocket 连接（`/ws` 或类似路径 → 容器 ttyd）
- 确认 `environment.ts` 返回的 `terminalUrl` 使用的是**相对路径**（如 `/ws?token=...`）而不是 `ws://localhost:PORT`
- 如果 `terminalUrl` 返回的是 `ws://host:port`，需要改为通过 Express 代理

**检查 `platform/src/components/Terminal.tsx`**：
- 确认 `wsUrl` 的使用方式
- 如果它直接拼接 `ws://host:port/ws`，需要改为 `wss://byocc.cc/ws?...`

**验证 Cloudflare Tunnel 支持 WebSocket**：
- Cloudflare Tunnel 默认支持 WebSocket，无需额外配置
- 但需要确认 WebSocket upgrade 请求走的是 Express 代理路径

### 任务 5：前端 API 客户端的生产适配

**文件**：`platform/src/lib/api.ts`

检查所有 API 调用函数：
1. `apiUrl()` — `NEXT_PUBLIC_API_URL` 为空时使用相对路径 ✓（需确认）
2. `authorizedFetch()` — `credentials: "include"` — 在 HTTPS 跨域时需要 CORS 配置正确
3. `file-reader.ts` 的 `apiUrl()` — 同上

**需要确认**：
- `NEXT_PUBLIC_API_URL` 在生产环境中**不设置**（留空），让浏览器走 `https://byocc.cc/api/*`
- 如果有 `.env.production` 或构建时注入的环境变量，确保没有硬编码 localhost

### 任务 7：启动脚本

创建 `start-production.sh`（或 `.ps1`）用于一键启动所有服务：

```powershell
# start-production.ps1
# BYOCC Production Startup Script

Write-Host "=== BYOCC Production Startup ===" -ForegroundColor Cyan

# 1. 确认 Docker 运行
Write-Host "Checking Docker..." -ForegroundColor Yellow
docker ps | Out-Null
if ($LASTEXITCODE -ne 0) {
    Write-Host "Docker is not running. Please start Docker Desktop." -ForegroundColor Red
    exit 1
}

# 2. 构建 server
Write-Host "Building server..." -ForegroundColor Yellow
Set-Location server
npm run build
if ($LASTEXITCODE -ne 0) { exit 1 }

# 3. 启动 server（后台）
Write-Host "Starting server on port 3001..." -ForegroundColor Yellow
Start-Process -NoNewWindow -FilePath "node" -ArgumentList "dist/index.js" -WorkingDirectory $PWD
Start-Sleep -Seconds 2

# 4. 构建 platform
Write-Host "Building platform..." -ForegroundColor Yellow
Set-Location ..\platform
npm run build
if ($LASTEXITCODE -ne 0) { exit 1 }

# 5. 启动 platform（后台）
Write-Host "Starting Next.js on port 3000..." -ForegroundColor Yellow
Start-Process -NoNewWindow -FilePath "npm" -ArgumentList "run", "start" -WorkingDirectory $PWD
Start-Sleep -Seconds 2

# 6. 健康检查
Write-Host "Health check..." -ForegroundColor Yellow
$response = Invoke-WebRequest -Uri "http://127.0.0.1:3001/api/health" -UseBasicParsing
Write-Host "Server: $($response.Content)" -ForegroundColor Green

Write-Host ""
Write-Host "=== BYOCC is running ===" -ForegroundColor Green
Write-Host "  Local:  http://localhost:3000" -ForegroundColor White
Write-Host "  Public: https://byocc.cc (via Cloudflare Tunnel)" -ForegroundColor White
Write-Host ""
Write-Host "Make sure cloudflared tunnel is running." -ForegroundColor Yellow
```

```bash

cd server && npm run build && node dist/index.js
cd platform && npm run build && npm run start

# 2. 本地测试
curl http://127.0.0.1:3001/api/health
# → {"status":"ok",...}

curl http://127.0.0.1:3000/api/health
# → 同上（通过 Next.js rewrite）
```

### Cloudflare Tunnel 验证

```bash
# 安装 cloudflared（如果还没装）
# 参照 https://developers.cloudflare.com/cloudflare-one/connections/connect-networks/downloads/

# 临时测试（不需要域名配置）：
cloudflared tunnel --url http://localhost:3000
# 会输出一个 https://xxx.trycloudflare.com 地址
# 用这个地址在浏览器里访问，确认 HTTPS + API + 终端都正常

# 正式配置（绑定 byocc.cc）：
# 1. cloudflared tunnel login
# 2. cloudflared tunnel create byocc
# 3. cloudflared tunnel route dns byocc byocc.cc
# 4. cloudflared tunnel run byocc
```

### 线上验证清单

1. `https://byocc.cc` → 首页正常加载
2. `https://byocc.cc/api/health` → 返回 `{"status":"ok"}`
3. 登录流程正常（cookie 在 HTTPS 下正确设置）
4. 进 Lab → 启动容器 → 编辑代码 → 提交 → 构建 → 终端运行
5. 终端 WebSocket 连接正常（`wss://byocc.cc/...`）
6. 只读文件查看正常
7. `docker inspect <container>` → ENV 中无真实 API Key
8. 浏览器 DevTools → 无混合内容（HTTP/HTTPS 混用）警告
9. 尝试 `http://byocc.cc` → 自动跳转 HTTPS（Cloudflare 默认开启）
