# 前端多文件编辑开发提示词

## 项目背景

BYOCC (Build Your Own Claude Code) 教学平台。学习者在浏览器中编辑 Docker 容器内的代码文件，提交构建并在容器内运行。

当前编辑器只能编辑**单个文件**（一个 Lab 一个 `code` 字符串），文件树组件存在但点击无效果。需要改造为**多文件编辑**模式，支持 Lab 0 的 6 文件编辑场景。

## 必须阅读的文件

开始前先读这些文件理解当前实现：

```
platform/src/components/LabRightArea.tsx   — 主工作区，管理编辑器和文件树
platform/src/components/FileTree.tsx        — 文件树组件（点击事件已有但被忽略）
platform/src/components/CodeEditor.tsx      — Monaco 编辑器封装
platform/src/lib/file-tree-data.ts          — 文件树静态数据 + Lab 可编辑文件映射
platform/src/lib/labs.ts                    — Lab 定义、骨架代码、文件名
platform/src/lib/api.ts                     — API 客户端（authorizedFetch、workspace API）
```

## 当前架构（需要改造的部分）

### 数据流（单文件）
```
workspace API (GET/PUT /api/labs/:id/workspace)
  → 存储/返回单个 code 字符串
  → LabRightArea 用一个 state: code
  → CodeEditor 接收 code + onChange
  → 自动保存回 workspace API
```

### 文件树
```
file-tree-data.ts 定义了 FILE_TREE（静态树结构）和 LAB_EDITABLE_FILES（每个 Lab 可编辑的文件路径）
→ FileTree 组件渲染树，点击可编辑文件调用 onFileSelect(path)
→ 但 LabRightArea 传的 onFileSelect 是空函数 () => {}
```

### CodeEditor
```
props: code, fileName, onChange
→ 没有 readOnly、activeFile 概念
→ 语言固定为 typescript
```

## 目标架构（多文件）

### 数据流（多文件）
```
workspace API 扩展：
  GET  /api/labs/:id/workspace  → 返回 { files: { [path]: code }, updatedAt }
  PUT  /api/labs/:id/workspace  → 接收 { files: { [path]: code } }

文件读取 API（后端队友开发，你不需要写）：
  GET  /api/files/:path?sessionId=xxx  → 从容器读取文件内容
  返回：{ content: string, language: string }

LabRightArea 状态改为：
  files: Map<string, string>     — 每个文件路径 → 内容
  activeFilePath: string          — 当前编辑的文件路径
  originalFiles: Map<string, string> — 容器中的原始内容（用于只读文件）
```

### 交互流程
```
用户进入 Lab 页面
  ↓
bootstrap:
  1. 加载 workspace（学生已保存的文件）
  2. 加载骨架代码作为默认值
  ↓
用户点击文件树中的文件
  ├── 可编辑文件（⭐）:
  │   ├── workspace 中有已保存的内容 → 加载它
  │   └── workspace 中没有 → 加载骨架代码
  │   → 编辑器可编辑
  │
  └── 只读文件（🔒）:
      → 调 GET /api/files/:path → 显示容器中的原始文件
      → 编辑器设为 readOnly
  ↓
编辑文件 → 自动保存（防抖 1.5s）
  → 保存整个 files map 到 workspace API
  ↓
提交代码 → 把所有可编辑文件的内容提交到容器
```

## 具体开发任务

### 任务 1：修改 workspace API 类型定义

**文件**：`platform/src/lib/api.ts`

当前 workspace 相关函数：

```typescript
export async function getWorkspace(labId: number): Promise<{ code: string | null; updatedAt: string | null }>
export async function saveWorkspace(labId: number, code: string): Promise<{ code: string; updatedAt: string }>
```

需要改为支持多文件：

```typescript
export type WorkspaceFiles = Record<string, string>; // path → code

export async function getWorkspace(labId: number): Promise<{
  files: WorkspaceFiles;
  updatedAt: string | null;
}>
export async function saveWorkspace(labId: number, files: WorkspaceFiles): Promise<{
  files: WorkspaceFiles;
  updatedAt: string;
}>
```

**重要**：要兼容后端还没改的情况。如果后端仍然返回 `{ code: string }` 格式，前端需要兼容处理：

```typescript
// 兼容旧版单文件格式
const data = await response.json();
if (typeof data.code === 'string' && !data.files) {
  // 旧格式：把 code 放入默认文件路径
  return { files: { [defaultPath]: data.code }, updatedAt: data.updatedAt ?? null };
}
return { files: data.files ?? {}, updatedAt: data.updatedAt ?? null };
```

### 任务 2：新增文件读取 API 客户端

**文件**：`platform/src/lib/api.ts`

```typescript
export async function readFileFromContainer(
  filePath: string,
  sessionId: string,
): Promise<{ content: string; language: string }> {
  const response = await authorizedFetch(
    apiUrl(`/api/files/${encodeURIComponent(filePath)}?sessionId=${encodeURIComponent(sessionId)}`),
  );
  if (!response.ok) {
    throw new Error(`Failed to read file: ${filePath}`);
  }
  return response.json();
}
```

注意：文件读取需要容器正在运行。如果容器没启动，这个 API 会失败——这时显示提示而不是崩溃。

### 任务 3：修改 CodeEditor 支持 readOnly 和多语言

**文件**：`platform/src/components/CodeEditor.tsx`

当前 props：
```typescript
type CodeEditorProps = {
  code: string;
  fileName: string;
  onChange: (value: string) => void;
};
```

改为：
```typescript
type CodeEditorProps = {
  code: string;
  fileName: string;
  readOnly?: boolean;
  onChange: (value: string) => void;
};
```

修改点：
1. `readOnly` prop 传入 Monaco 的 `options.readOnly`
2. 根据文件扩展名推断语言（不要硬编码 `typescript`）：
   ```typescript
   function inferLanguage(fileName: string): string {
     if (fileName.endsWith('.tsx')) return 'typescript';
     if (fileName.endsWith('.ts')) return 'typescript';
     if (fileName.endsWith('.json')) return 'json';
     if (fileName.endsWith('.md')) return 'markdown';
     if (fileName.endsWith('.mjs')) return 'javascript';
     return 'plaintext';
   }
   ```
3. 顶部文件名标签显示只读状态：
   ```tsx
   {readOnly && <span className="text-[10px] text-[var(--text-muted)]">🔒 只读</span>}
   ```
4. `readOnly` 模式下隐藏 minimap、降低行高、用稍浅的背景色

### 任务 4：重构 LabRightArea 为多文件编辑

**文件**：`platform/src/components/LabRightArea.tsx`

这是最大的改动。核心变化：

#### 4.1 状态改造

```typescript
// 删掉：
const [code, setCode] = useState<string>("");

// 改为：
const [files, setFiles] = useState<Record<string, string>>({});
const [activeFilePath, setActiveFilePath] = useState<string>("");
const [readOnlyContent, setReadOnlyContent] = useState<string | null>(null);
const [readOnlyLanguage, setReadOnlyLanguage] = useState<string>("typescript");
const [isLoadingFile, setIsLoadingFile] = useState(false);
```

#### 4.2 bootstrap 改造

```typescript
async function bootstrap() {
  // ... session 创建保持不变 ...

  const workspace = await getWorkspace(lab.id);
  if (!cancelled) {
    setFiles(workspace.files);
    // 设置默认活跃文件为第一个可编辑文件
    const editableFiles = LAB_EDITABLE_FILES[lab.id] ?? [];
    if (editableFiles.length > 0) {
      const firstFile = editableFiles[0];
      setActiveFilePath(firstFile);
      // 如果 workspace 没有这个文件的内容，用骨架代码
      if (!workspace.files[firstFile]) {
        setFiles(prev => ({
          ...prev,
          [firstFile]: LAB_SKELETONS[lab.id] ?? "",
        }));
      }
    }
  }
}
```

#### 4.3 文件切换逻辑

```typescript
const handleFileSelect = async (filePath: string) => {
  const editableSet = new Set(LAB_EDITABLE_FILES[lab.id] ?? []);
  const isEditable = editableSet.has(filePath);

  if (isEditable) {
    // 可编辑文件：切换到文件编辑模式
    setReadOnlyContent(null);
    setActiveFilePath(filePath);
    // 如果这个文件还没有加载过，用骨架代码
    if (!files[filePath] && files[filePath] !== "") {
      // Lab 0 场景：骨架代码可能来自 LAB_SKELETONS 或默认空字符串
      setFiles(prev => ({
        ...prev,
        [filePath]: prev[filePath] ?? "",
      }));
    }
  } else {
    // 只读文件：从容器读取
    if (!sessionId || environmentStatus !== "running") {
      setEnvironmentMessage("请先启动实验环境才能查看只读文件。");
      return;
    }
    setIsLoadingFile(true);
    try {
      const result = await readFileFromContainer(filePath, sessionId);
      setActiveFilePath(filePath);
      setReadOnlyContent(result.content);
      setReadOnlyLanguage(result.language);
    } catch {
      setEnvironmentMessage("无法读取文件，请确认实验环境已启动。");
    } finally {
      setIsLoadingFile(false);
    }
  }
};
```

#### 4.4 代码修改处理

```typescript
const handleCodeChange = (value: string) => {
  if (!activeFilePath) return;
  setFiles(prev => ({
    ...prev,
    [activeFilePath]: value,
  }));
  setSaveState("dirty");
};
```

#### 4.5 自动保存改造

```typescript
useEffect(() => {
  if (saveState !== "dirty") return;
  const timer = window.setTimeout(() => {
    setSaveState("saving");
    void saveWorkspace(lab.id, files)
      .then((workspace) => {
        setLastSavedAt(workspace.updatedAt);
        setSaveState("saved");
      })
      .catch(() => {
        setSaveState("error");
      });
  }, 1500);
  return () => window.clearTimeout(timer);
}, [files, lab.id, saveState]);
```

#### 4.6 传递给 CodeEditor

```tsx
<CodeEditor
  code={readOnlyContent ?? files[activeFilePath] ?? ""}
  fileName={activeFilePath}
  readOnly={readOnlyContent !== null}
  onChange={handleCodeChange}
/>
```

#### 4.7 传递给 FileTree

```tsx
<FileTree
  labId={lab.id}
  activeFilePath={activeFilePath}
  onFileSelect={(path) => { void handleFileSelect(path); }}
/>
```

#### 4.8 提交代码改造

```typescript
const handleSubmit = async () => {
  // ... 环境检查保持不变 ...
  try {
    setBuildState("building");
    await saveCurrentWorkspace();
    // 提交时传所有可编辑文件
    const editableFiles = LAB_EDITABLE_FILES[lab.id] ?? [];
    for (const filePath of editableFiles) {
      const content = files[filePath];
      if (content !== undefined) {
        await submitCode(sessionId, content, lab.id, filePath);
      }
    }
    // ... 或者如果 submitCode 只接受单个文件，改后端接口
  }
};
```

**注意**：当前的 `submitCode` 只接受一个 `code` 字符串。后端队友可能需要改接口来支持多文件提交。你需要看 `submitCode` 的当前签名，如果后端还没改，先兼容单文件提交（只提交 activeFile 或第一个可编辑文件）。

## Lab 0 特殊处理

Lab 0 跟其他 Lab 不同：
- 其他 Lab（1-5）：编辑**一个文件**，骨架代码预填充
- Lab 0：编辑**6 个文件**，内容从容器读取

Lab 0 的 bootstrap 流程需要特殊处理：
1. 启动环境（容器）
2. 环境启动后，调文件读取 API 获取 6 个文件的原始内容
3. 如果 workspace 有已保存的文件 → 用已保存的
4. 如果没有 → 用容器里的原始文件作为初始内容

这意味着 Lab 0 的文件内容需要容器先启动才能加载。在容器启动前，编辑器可以显示提示 "启动实验环境后加载文件内容"。
