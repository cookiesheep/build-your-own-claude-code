> 给新的 Claude Code 会话或队友使用。
> 工作目录：D:\code\build-your-own-claude-code
> 这是一个纯内容编写任务，不涉及代码逻辑。

---

## 任务概述

为 BYOCC 平台的文件目录树编写每个文件/目录的中文描述。这些描述将在用户鼠标悬浮在文件上时作为 tooltip 显示，帮助学习者理解 Claude Code 各模块的作用。

**产出物**：`platform/src/lib/file-descriptions.ts`（导出 TypeScript 对象）

---

## 项目背景

**Build Your Own Claude Code** 让学习者通过 6 个 Lab 逐步实现 Claude Code 的核心模块：

| Lab | 主题 | 学习者实现什么 |
|-----|------|--------------|
| 0 | 环境体验 | 安装运行完整 Claude Code |
| 1 | 消息协议 | messages.ts — LLM 对话的数据结构 |
| 2 | 工具系统 | read-file / write-file / bash — 给 Agent 装手脚 |
| **3** | **Agent Loop** | **query.ts — while(true) 循环** |
| 4 | 规划能力 | subagent / todo-write — 先想再做 |
| 5 | 上下文压缩 | compress — 长对话不崩 |

这些文件在平台的文件目录树中展示。学习者需要知道每个文件是干什么的、跟哪个 Lab 相关。

## 编写要求

### 质量标准

1. **brief**：20-40 字，一句话说清"这是什么"。避免废话（不要写"这是一个..."）。
2. **detail**：50-100 字，解释文件在 Claude Code 系统中的作用。要包含：
   - 这个文件解决什么问题
   - 它的输入/输出是什么
   - 与其他模块的协作关系（如果有的话）
3. **labs**：标注该文件与哪些 Lab 直接相关。核心文件可能关联多个 Lab。
4. **tag**：正确分类，前端可能按 tag 显示不同颜色。

### 语气要求

- 用**第二人称**（"你将在这里实现..."，"你学到的是..."）
- 技术术语保留英文（Agent Loop, tool_use, Harness）
- 避免过度简化——学习者是有一定编程基础的大学生
- 关键文件（query.ts, messages.ts, tools/*）的描述可以更详细

### Claude Code 架构背景（写描述时参考）

```
Claude Code 的架构分层：

1. 消息层（messages.ts）
   定义 Message, ContentBlock 类型。
   LLM 的输入输出都是结构化消息，不是原始字符串。

2. 工具层（tools/*）
   每个工具是一个 { name, description, inputSchema, execute } 对象。
   registry.ts 管理工具注册，给 LLM 提供 tools 列表。

3. 循环层（query.ts）
   Agent Loop: while(true) { callLLM → parseToolUse → execute → feedBack }
   这是 Agent 的核心——没有循环，LLM 只能说话不能做事。

4. 规划层（agent/*）
   subagent: 把复杂任务拆成子任务，每个子任务有自己的 Agent Loop。
   todo-write: 让 Agent 在行动前列出计划。

5. 上下文层（context/*）
   长对话时 token 会超限，需要压缩历史消息。
   compress.ts 实现三层策略：滑动窗口 + 摘要 + 关键信息保留。

6. 基础设施（core/*）
   config.ts: 全局配置（模型选择、超时、并发限制）。
   permissions.ts: 工具权限控制（哪些操作需要用户确认）。

7. 入口（cli.ts）
   解析命令行参数，初始化配置，启动 Agent Loop，连接 TUI。
```

### 每个条目的描述重点

| 文件 | 描述重点 |
|------|---------|
| src/query.ts | 强调这是**最核心的文件**，while(true) 循环，Lab 3 的主角 |
| src/messages.ts | 消息不是字符串而是结构化的，四种 ContentBlock 类型 |
| src/tools/* | 每个"工具"是一个标准化对象，registry 统一管理 |
| src/agent/subagent.ts | 子 Agent 有自己的循环，像递归调用 |
| src/agent/todo-write.ts | 让 Agent 先规划再执行，类似人类列 Todo |
| src/context/compress.ts | 三层压缩策略，解决 token 上限问题 |
| src/context/summarize.ts | 把长对话压缩成摘要 |
| src/core/config.ts | 全局配置中心 |
| src/core/permissions.ts | 安全边界，危险操作需确认 |
| src/cli.ts | 从命令行到 Agent Loop 的桥梁 |
| shared/types.ts | 全局类型定义，所有模块共享 |
| build.mjs | 构建 Lab 代码的脚本，连接学习者代码和完整系统 |

  // 收集所有路径
  function collectPaths(nodes) {
    const paths = [];
    for (const node of nodes) {
      paths.push(node.path);
      if (node.children) paths.push(...collectPaths(node.children));
    }
    return paths;
  }
  const allPaths = collectPaths(FILE_TREE);
  const descPaths = new Set(FILE_DESCRIPTIONS.map(d => d.path));
  const missing = allPaths.filter(p => !descPaths.has(p));
  if (missing.length > 0) {
    console.log('缺少描述的条目:', missing);
  } else {
