**Build Your Own Claude Code (BYOCC)** 是一个基于真实 Claude Code 源码的渐进式教学平台。

当前前端状态：
- 营销首页 `/` 已完成（Canvas 粒子动画 + 琥珀金配色 + 深色主题）
- Lab 选择页 `/platform` 已完成
- 实验工作台 `/lab/:id` 已完成（可调整面板 + 文件目录树 + Monaco 编辑器 + xterm.js 终端）
- 后端认证系统正在开发中（用户名/密码 + JWT cookie）

**本次任务**：实现认证相关的前端页面和交互——**这是一个可以炫技的页面**。

## 设计定位

### 核心原则

**这个登录页面是用户对平台的第一印象。它需要让人记住。**

- 这是一个"showcase"级别的页面，不是普通的表单
- 动效和交互效果由你自主设计和实现
- **不要复用首页的粒子动画**——要有自己的视觉语言
- 可以大胆尝试新颖的效果，但要保证登录流程的可用性不受影响
- 配色体系与首页保持一致（见下方），但视觉表现可以完全不同

### 设计研究要求

**在开始编码之前，你必须先做设计研究：**

1. **搜索 2025-2026 年最新的登录页面设计趋势**
   - 搜索关键词建议："best login page design 2025"、"creative login page animation"、"modern auth UI 2026"
   - 关注 WebGL shader effects、fluid animation、magnetic interactions、kinetic typography 等前沿效果

2. **参考小红书作者「思花」的特效风格**
   - 如果你能访问小红书（xiaohongshu.com），搜索用户「思花」的作品
   - 她的前端特效特点是：流畅、有质感、交互感强、不花哨但高级
   - 思考如何将类似的质感融入登录场景

3. **研究至少 3 种不同方向的登录页动效方案**
   - 从搜索结果中提炼出 3 个你觉得最适合 BYOCC 的方向
   - 选择其中一个开始实现（如果效果不理想可以换方向）

4. **效果必须新颖**
   - 不要用以下烂大街效果：打字机动画、粒子连线、3D 旋转体、模糊光晕 blob、简单的 fadeIn
   - 要找 2024 年之后流行的新效果

## 页面结构

### 路由

```
/login          — 登录页面（独立全屏页面）
```

### 登录页布局

**不要做居中白色卡片的普通登录页。布局由你设计。**

可能的布局方向（仅供参考，不是限制）：
- 全屏沉浸式（背景动效 + 浮动表单）
- 分屏式（左侧品牌展示 + 右侧表单）
- 画布式（整个页面是交互画布，表单融入其中）
- 其他你认为更好的方案

### 表单元素

必须包含的元素：
- 标题/品牌名（"BYOCC" 或 "Build Your Own Claude Code"）
- 用户名输入框
- 密码输入框（带显示/隐藏切换）
- 登录按钮
- 错误提示区域
- 主题切换按钮

可选元素：
- "记住我" 复选框
- 装饰性文字或标语
- 品牌动画

### 交互细节

**表单交互必须精致：**
- 输入框 focus 时有动效（不是简单的边框变色——要有设计感）
- 密码错误时卡片/表单有反馈动画（shake、glitch、color shift 等）
- 登录按钮 loading 状态（不是简单的 spinner）
- 登录成功后有过渡动画再跳转（不要生硬跳转）
- 鼠标 hover 时表单元素有微交互

## 后端 API 对接

后端认证系统正在并行开发。以下是接口约定（前端按这个契约开发，后端会保证接口一致）：

### POST /api/auth/login

```
请求：
{
  "username": "alice",
  "password": "password123"
}

成功（200）：
{
  "success": true,
  "user": {
    "id": "uuid-xxx",
    "username": "alice",
    "role": "user"
  }
}
注意：成功时后端会 Set-Cookie: byocc_session=JWT（httpOnly）
前端不需要手动存储 token，浏览器会自动带 cookie

失败（401）：
{
  "success": false,
  "error": "用户名或密码错误"
}
```

### GET /api/auth/me

```
已登录（200）：
{
  "authenticated": true,
  "user": { "id": "uuid", "username": "alice", "role": "user" }
}

未登录（401）：
{
  "authenticated": false
}
```

### POST /api/auth/logout

```
成功（200）：
{
  "success": true
}
清除 cookie
```

### 后端地址

- 开发环境：`http://localhost:3001`
- 前端 dev server 需要配置 proxy 或在 Next.js `next.config.js` 中配置 rewrites
- 所有认证请求需要 `credentials: 'include'` 以携带 cookie

### Next.js 代理配置

如果还没有配置，在 `platform/next.config.mjs` 或 `next.config.js` 中添加：

```javascript
// next.config.mjs
const nextConfig = {
  async rewrites() {
    return [
      {
        source: '/api/:path*',
        destination: 'http://localhost:3001/api/:path*',
      },
    ];
  },
};
```

## 认证状态管理

### 新建 `platform/src/lib/auth.ts`

```typescript
// 认证状态管理（客户端）

export interface User {
  id: string;
  username: string;
  role: string;
}

export interface AuthState {
  isAuthenticated: boolean;
  user: User | null;
  loading: boolean;
}

// checkAuth(): Promise<AuthState>
//   GET /api/auth/me → 返回认证状态
//   页面加载时调用

// login(username: string, password: string): Promise<{ success: boolean; user?: User; error?: string }>
//   POST /api/auth/login → 登录

// logout(): Promise<void>
//   POST /api/auth/logout → 登出 → 跳转 /
```

## 技术约束

### 必须遵守

- Next.js 14 App Router
- TypeScript strict
- Tailwind CSS（与现有项目一致）
- 已有的 CSS 变量（见 `platform/src/app/globals.css`）
- `data-theme` 属性控制明暗模式
- 所有 fetch 请求带 `credentials: 'include'`

### 可以自由选择

- 动效库（GSAP、Framer Motion、或纯 CSS/Canvas）
- 背景效果实现方式（Canvas、WebGL、SVG、CSS）
- 布局方案
- 表单交互细节
- 过渡动画方案

### 不需要新装大型依赖

- 优先使用 CSS 动画 + 手写 JS
- 如果确实需要动画库，优先 Framer Motion（React 生态标准）
- 不要为了一个小效果引入一个大型库

## 关键提醒

1. **先研究再动手**——花 10 分钟搜索设计趋势比盲目编码产出好 10 倍
2. **配色是约束，效果是自由**——琥珀金配色不变，但视觉表达可以天马行空
3. **可用性 > 炫技**——再酷的效果也不能让用户找不到登录按钮
4. **不要复用首页粒子**——登录页要有自己的视觉语言
5. **后端可能还没就绪**——先用 mock 响应开发，确保 API 调用代码正确即可
6. **如果你用 /frontend-design skill 或 designer agent，直接把这个文件喂给它**
