## 项目背景

**Build Your Own Claude Code (BYOCC)** 是一个基于真实 Claude Code 源码的渐进式教学平台。

当前状态：
- 后端已完成 12 个 PR，所有核心 API 可用（Express + Docker + SQLite）
- 前端已有营销首页 `/` + Lab 选择页 `/platform` + 实验工作台 `/lab/:id`
- 当前使用匿名身份系统（`POST /api/auth/anonymous` 生成随机 userId）
- 平台即将公开部署到 byocc.cc

**本次任务**：添加用户名/密码认证系统，替代匿名身份，保护容器资源。

## 认证方案

### 用户模型

```
管理员（Leader）通过 CLI 脚本创建用户账号。
前期不开放公开注册，只有获得账号的人才能使用实验工作台。
后续（Phase 3）会加入 GitHub OAuth 开放注册。
```

### 认证流程

```
1. 管理员运行 CLI: npx tsx server/src/scripts/create-user.ts --username alice --password xxx
2. 用户访问 /platform → 点击 Lab → 检测未登录 → 跳转 /login
3. 用户在 /login 输入用户名+密码 → POST /api/auth/login
4. 后端验证密码 → 返回签名 cookie (httpOnly, secure, sameSite=strict)
5. 前端获取 GET /api/auth/me → 显示用户名 + 登出按钮
6. 后续所有 API 请求自动携带 cookie
7. /platform 和 / 首页无需登录即可浏览
8. /lab/:id 需要 cookie 认证才能访问
```

### 密码安全

```
- 使用 bcrypt 哈希（npm bcryptjs，纯 JS 实现，不需要 native 编译）
- salt rounds: 10
- 密码最短 6 位（CLI 创建时校验）
- 数据库只存哈希，不存明文
```

### Cookie 配置

```
名称: byocc_session
内容: JWT token（包含 userId, username, exp）
签名密钥: SERVER_SESSION_SECRET 环境变量
属性: httpOnly=true, secure=true(生产)/false(开发), sameSite=strict, path=/, maxAge=7d
```

## 数据库改动

### users 表

```sql
CREATE TABLE IF NOT EXISTS users (
  id TEXT PRIMARY KEY,           -- UUID
  username TEXT UNIQUE NOT NULL,
  password_hash TEXT NOT NULL,
  role TEXT DEFAULT 'user',      -- 'admin' 或 'user'
  created_at TEXT DEFAULT (datetime('now'))
);
```

### 修改 sessions 表

```sql
-- sessions 表的 user_id 字段从匿名 UUID 改为关联 users.id
-- 匿名身份系统保留作为降级方案，但优先使用 cookie 认证
ALTER TABLE sessions ADD COLUMN auth_method TEXT DEFAULT 'anonymous';
```

### user_settings 表（API Key 用，本次一并创建）

```sql
CREATE TABLE IF NOT EXISTS user_settings (
  user_id TEXT PRIMARY KEY,
  api_key_encrypted TEXT,        -- AES 加密的 API Key
  api_key_source TEXT DEFAULT 'default',  -- 'default' 或 'user'
  updated_at TEXT DEFAULT (datetime('now')),
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

## 与匿名身份系统的关系

```
保留 POST /api/auth/anonymous 作为降级方案：
- 如果前端检测到 cookie 有效 → 用 cookie 认证
- 如果没有 cookie 且不在 /lab/:id 页面 → 不需要认证
- /lab/:id 页面必须有 cookie 认证

sessions 表的 user_id 字段：
- cookie 认证时 → user_id = users.id
- 降级匿名时 → user_id = 随机 UUID（保持兼容）
```
